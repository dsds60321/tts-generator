# 요구사항: 한국어 TTS 프로토타입 서비스

**정의일:** 2026-03-21
**핵심 가치:** 사용자가 한국어 텍스트를 입력하고 원하는 화자와 속도를 선택했을 때, 자연스러운 음질의 `WAV` 음성을 안정적으로 즉시 생성해 다운로드할 수 있어야 한다.

## v1 요구사항

### API

- [ ] **API-01**: API 사용자는 `/health`를 호출해 서비스 상태를 JSON으로 확인할 수 있다
- [ ] **API-02**: API 사용자는 `/voices`를 호출해 선택 가능한 한국어 화자 목록을 `id`와 표시명과 함께 받을 수 있다
- [ ] **API-03**: API 사용자는 `/synthesize`에 텍스트, 화자, 속도를 보내고 선택값이 반영된 `WAV` 파일을 HTTP 응답으로 받을 수 있다

### 웹 경험

- [ ] **WEB-01**: 사용자는 한 페이지 웹 화면에서 텍스트 입력, 화자 선택, 속도 선택, 생성 실행을 한 번에 할 수 있다
- [ ] **WEB-02**: 사용자는 웹 화면에서 음성 생성을 요청하고 결과 `WAV` 파일을 바로 다운로드할 수 있다
- [ ] **WEB-03**: 사용자는 웹 화면에서 생성 중 상태와 실패 메시지를 읽기 쉬운 형태로 볼 수 있다

### 합성 제어

- [ ] **SYN-01**: 사용자는 한국어 텍스트를 입력해 음성 생성을 요청할 수 있다
- [ ] **SYN-02**: 사용자는 지원되는 한국어 화자 중 하나를 선택할 수 있다
- [ ] **SYN-03**: 사용자는 허용된 범위 안에서 음성 속도를 조절할 수 있다

### 검증

- [ ] **VAL-01**: 빈 문자열 또는 공백만 있는 텍스트 요청은 읽기 쉬운 메시지와 함께 `4xx`로 거절된다
- [ ] **VAL-02**: 지원되지 않는 화자 값은 읽기 쉬운 메시지와 함께 `4xx`로 거절된다
- [ ] **VAL-03**: 허용 범위를 벗어나거나 형식이 잘못된 속도 값은 읽기 쉬운 메시지와 함께 `4xx`로 거절된다

### 확장성

- [ ] **ARCH-01**: 개발자는 이후 phase에서 내부 TTS 엔진을 교체하더라도 `/synthesize`의 공개 요청 필드와 `WAV` 응답 계약을 유지할 수 있다

## v2 요구사항

### 긴 텍스트

- **LONG-01**: 사용자는 phase 1 길이 제한을 넘는 긴 텍스트를 넣어도 자동 분할/병합으로 음성을 생성할 수 있다

### 생성 이력

- **HIST-01**: 사용자는 최근 생성 이력을 보고 이전 결과를 다시 다운로드할 수 있다

### 비동기 처리

- **ASYN-01**: 사용자는 오래 걸리는 음성 생성 작업을 비동기로 시작하고 완료 상태를 확인할 수 있다

### 제공자 관리

- **PROV-01**: 운영자는 공개 API 계약을 유지한 채 네트워크 기반 엔진과 로컬 엔진 중 하나를 설정으로 선택할 수 있다

## 범위 제외

현재 로드맵에서 명시적으로 제외한 항목이다.

| Feature | Reason |
|---------|--------|
| DB 저장 | 프로토타입 단계에서는 즉시 다운로드 흐름 검증이 더 중요하다 |
| 인증/권한 관리 | 초기 체험형 제품 범위에 비해 복잡도가 과하다 |
| 결제 | 핵심 가치 검증 전에 수익화 범위를 늘리지 않는다 |
| 관리자 기능 | 운영 도구보다 사용자 생성 흐름 검증이 우선이다 |
| 작업 큐/백그라운드 워커 | phase 1 최소 구조와 충돌하므로 후속 단계로 미룬다 |
| 배포 자동화 | 로컬 실행 검증이 먼저다 |
| 다국어 지원 | v1은 한국어 품질 검증에 집중한다 |
