# 한국어 TTS 프로토타입 서비스

## What This Is

웹 UI와 API를 통해 한국어 텍스트를 음성으로 변환하고 `WAV` 파일로 바로 다운로드할 수 있는 프로토타입 서비스다. 일반 사용자도 브라우저에서 바로 써볼 수 있어야 하며, 개발자는 `FastAPI` 기반 API로 동일한 기능을 호출할 수 있어야 한다. 첫 구현은 최소 파일 구조로 빠르게 전체 흐름을 검증하되, 이후 화자 선택, 속도 조절, 이력 관리, 긴 텍스트 분할, 비동기 처리, 엔진 교체가 가능하도록 방향을 잡는다.

## Core Value

사용자가 한국어 텍스트를 입력하고 원하는 화자와 속도를 선택했을 때, 자연스러운 음질의 `WAV` 음성을 안정적으로 즉시 생성해 다운로드할 수 있어야 한다.

### Out of Scope

- DB 저장 기능 — 프로토타입 단계에서는 실행 가능성과 생성 흐름 검증이 우선이다.
- 인증과 권한 관리 — 일반 사용자 체험용 초기 버전에서는 핵심 가치와 직접 관련이 없다.
- 결제 기능 — 프로토타입 단계에서 수익화 기능은 범위를 과도하게 키운다.
- 작업 큐와 복잡한 비동기 인프라 — 초기에는 즉시 생성 흐름을 먼저 검증한다.
- 관리자 기능 — 운영용 기능보다 사용자 생성 경험 검증이 먼저다.
- 배포 자동화 — 로컬 실행과 기능 검증이 우선이며 CI/CD는 이후 단계로 미룬다.

## Context

이 프로젝트는 Python 기반으로 구현하며, 웹 서비스 프레임워크는 `FastAPI`를 사용한다. 첫 phase는 `main.py` 단일 파일 또는 그에 가까운 최소 구조에서 시작해야 하며, 불필요하게 디렉터리를 세분화하지 않는다. 사용자는 웹 또는 API를 통해 한국어 텍스트를 넣고 음성 파일을 받아야 하므로, 브라우저에서 바로 사용할 수 있는 단일 페이지 UI와 다운로드 흐름이 필요하다. 실행 환경은 `macOS`와 `Linux`를 모두 고려해야 하며, 초기부터 자연스러운 한국어 음질이 중요하다. 초기 엔진은 무료이며 별도 서비스 키가 필요 없는 라이브러리 기반 구성을 선호한다.

## Constraints

- **Tech stack**: `Python` + `FastAPI` 사용 — 사용자가 명시한 기술 방향이다.
- **Architecture**: `main.py` 중심의 최소 구조로 시작 — 초기 검증 속도를 높이고 과설계를 피하기 위함이다.
- **Platform**: `macOS`와 `Linux`에서 실행 가능해야 함 — 개발 및 사용 환경 호환성을 확보해야 한다.
- **Audio quality**: 초기부터 자연스러운 한국어 음질이 중요함 — 단순 동작 확인만으로는 성공 기준을 충족하지 못한다.
- **TTS provider**: 무료이며 별도 서비스 키가 필요 없는 라이브러리 우선 — 초기 진입 장벽과 운영 복잡도를 낮추기 위함이다.
- **Delivery**: `/synthesize`는 `WAV` 파일을 즉시 내려줘야 함 — 사용자가 바로 결과를 받는 경험이 핵심이기 때문이다.
